# HorseMD v0.6.0 Release Notes

## 新功能

### 更聪明的斜杠菜单

在富文本中输入 `/` 后，可以使用中文、英文、常用别名、完整拼音或拼音首字母查找命令。

- `/bt` 可以匹配“标题”。
- `/bg` 可以匹配“表格”。
- `/wxlb` 可以匹配“无序列表”。
- `/java`、`/python`、`/mermaid` 可以直接插入已设置语言的代码块。
- 短关键词使用精确和前缀排序，减少无关结果；没有结果时菜单不会突然消失。

菜单使用 HorseMD 自身主题变量，在不同主题和窗口边界下保持可读、可滚动。

### 一个工作区添加多个文件夹

工作区现在是一个未命名的多根目录容器：

- “打开文件夹”改为向当前工作区添加路径，不再替换已经打开的文件夹。
- 每个根目录独立展示、监听、展开和移除。
- 根目录是工作区挂载点，不能被误拖动、重命名或删除真实磁盘目录。
- 空工作区只保留清晰的“添加文件夹”入口。
- “添加文件夹”和“新建文件夹”使用不同图标与分组，减少误操作。
- 在文件树空白区域右键或双击，也可以快速打开工作区操作菜单。

## 体验改进

### 大纲默认展示更合理

大纲默认保留文档前两层实际层级：`H1 > H2 > H3` 文档会直接显示 H1 和 H2，
更深层分支保持折叠；如果文档全部由 H1 组成，则所有标题都会显示。文档从 H2 开始或跳级时，
也按照真实层级关系计算，而不是死按标题数字处理。

### 源码查找可以真正定位

源码模式下使用 `Ctrl/Cmd+F`：

- 当前匹配会滚动到可见区域并尽量居中。
- 当前结果使用主题适配的高对比度高亮。
- 富文本和源码之间切换时，会重建对应视图的搜索结果和位置。
- 保持查找栏打开往返切换，不再出现“只有计数但不跳转”的情况。

## 关键修复

### 源码与富文本切换定位

继续强化源码与富文本双向切换：

- 编辑状态跟随可见光标，阅读状态保持当前视口。
- 表格文字、表格行内代码和 CodeMirror 代码块使用结构化位置映射。
- 重复词不再依赖全文关键词匹配作为主定位路径。
- 富文本编辑器在源码模式下继续保持挂载，避免大文档重新解析和远程图片重新布局。
- 仅查看模式切换不会触发未保存状态；只有真实源码编辑才同步回富文本。
- 源码光标加粗、加长并按最终 textarea 宽度测量，减少遮字和落入空白区域。

该链路已覆盖普通文档连续双向切换、表格、代码块，以及约 12 万字、183 张远程图片的真实大文档回归。

### 工作区和安全边界

- 会话恢复和 watcher 会过滤相对路径、受限根目录和无效路径。
- 外部链接只允许批准的协议，阻止危险或未知协议交给系统打开。
- 本地字体权限只服务于字体选择器，不再进行宽泛授权。

## 稳定性与维护

本版本完成了一轮行为保持型重构：

- `Editor.jsx` 收窄为 Crepe 生命周期编排层。
- 源码切换状态机提取到 `useSourceModeSwitch.js`。
- Workspace、Sidebar tree、Review 卡片和 Review Decoration 分别独立。
- 主进程的文档、文件系统、watcher、PDF 和安全逻辑按领域拆分。
- 新增源码映射、源码查找、模式切换、Review UI、文件系统、watcher、PDF 和安全测试。
- CI 在构建前运行核心回归测试。

这些调整不改变现有公开操作，目的是降低后续新增功能时破坏保存、dirty、编辑器生命周期和跨平台行为的风险。

## 发布前验证

- `npm run test:core`

- 源码/富文本双向链路、表格、代码块和真实大文档回归。
- 多根工作区添加、移除、重启恢复、右键菜单和外部变更监听。
- 所有主题下的斜杠菜单、源码查找高亮和源码光标。
- Windows、macOS 安装包启动与版本核验；Android 构建按发布范围决定是否附带。
