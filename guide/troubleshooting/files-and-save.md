# 文件与保存

<span class="version-badge">适用于 HorseMD v0.6.0</span>

## 工作区会复制文件吗

不会。桌面端添加文件夹只是把真实磁盘目录挂到文件树，重命名、移动和删除都会作用于真实文件。移除工作区根目录只取消显示，不删除目录。

## 未命名草稿在哪里

有内容且未保存的临时标签会随会话恢复，并保持“已修改”。它存放在 HorseMD 用户数据中，不是已经创建的 `.md` 文件。重要内容应使用 `Ctrl/Cmd+S` 保存到明确位置。

## 外部程序改了文件

HorseMD 监听已打开文件和工作区目录。磁盘变化会刷新文件树；当前标签没有未保存内容时可以同步新内容。若你正在编辑，应用不会直接用外部版本覆盖。

## Android 文件管理器看不到文档

Android 可能禁止普通文件管理器浏览应用目录。请在 HorseMD 文件树中打开，或使用“导出/分享”复制到公共目录。卸载前必须导出重要文档。

## 图片或附件移动后失效

相对路径以 Markdown 文件目录为基准。移动文档时应同时移动关联的图片文件夹和 `assets/`。在源码模式检查链接是否仍指向正确相对位置。

::: tip 反馈时提供路径类型
报告文件问题时说明是绝对路径、相对路径、`file:///` 链接、网络地址，还是移动端系统 URI；不要公开包含敏感信息的完整个人路径。
:::
