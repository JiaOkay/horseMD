# 富文本与源码模式

<span class="version-badge">适用于 HorseMD v0.6.0</span>

点击状态栏中的“富文本/源码”，或按 `Ctrl/Cmd+/` 切换模式。

![HorseMD 源码模式](/images/v0.6.0/source-mode.png)
<span class="image-caption">源码模式保留 Markdown 标记，适合检查语法、批量编辑或处理复杂结构。</span>

## 两种切换行为

HorseMD 会根据你是否正在编辑决定定位方式：

1. 光标在可视区域内：切换后跟随对应光标，保证光标仍然可见。
2. 没有可见光标，只是在滚动阅读：切换后保持当前阅读视口，不会突然跳到旧光标。

表格单元格、行内代码、代码块和重复文字使用结构化位置映射，不依赖简单关键词搜索。富文本编辑器在源码模式下保持挂载，因此切回时不会重新解析整篇文档或重新加载远程图片。

## 源码编辑与保存状态

只切换模式不会把标签标记为“已修改”。在源码文本框真正输入、删除或粘贴后，新内容才会同步回富文本并进入未保存状态。

源码模式支持 `Ctrl/Cmd+F` 查找，当前结果会高亮并滚动到可见区域。从富文本带着查找栏切换过来时，HorseMD 会为新视图重建匹配结果。

::: tip 大文档
超大文档可能默认使用快速源码模式。确认设备性能足够后，可以选择进入富文本；图片很多的文档首次渲染需要更多时间。
:::
