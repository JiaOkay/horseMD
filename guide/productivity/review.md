# 审阅与 CriticMarkup

<span class="version-badge">适用于 HorseMD v0.6.0</span>

HorseMD 使用可读的 CriticMarkup 保存审阅痕迹。它仍然是普通 Markdown 文本，不依赖私有数据库。

## 添加审阅标记

在富文本中选中文字，打开浮动工具栏的审阅菜单，可以选择：

| 类型 | 源码形式 |
| --- | --- |
| 新增 | `{++新增内容++}` |
| 删除 | `{--删除内容--}` |
| 替换 | `{~~旧内容~>新内容~~}` |
| 高亮并评论 | `{==高亮内容==}{>>评论<<}` |

新增和删除会以不同颜色显示；替换同时保留旧内容与新内容；带评论高亮会在正文边缘显示评论入口。

## 处理审阅卡片

点击审阅标记或评论入口打开卡片，可以接受、拒绝或移除标记。接受替换会保留新内容，拒绝则恢复旧内容。所有动作都会进入编辑历史，并触发正常的未保存状态。

## 在源码模式阅读

源码模式不会把 CriticMarkup 当作普通删除线误解析。语法符号会弱化，内容仍可阅读；把光标移入标记时可以查看和直接修改原始语法。

::: warning 保持单条标记结构完整
不要随意删除一半的 `{++ ++}`、`{-- --}` 或替换分隔符。需要取消审阅时优先使用审阅卡片的接受、拒绝或移除按钮。
:::
