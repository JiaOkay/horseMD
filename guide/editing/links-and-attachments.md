# 链接与附件

<span class="version-badge">适用于 HorseMD v0.6.0</span>

## 打开链接

在富文本中按住 `Ctrl` 或 `Cmd` 点击链接：

- `https://` 和 `http://` 使用系统默认浏览器打开。
- `file:///` 本地文件链接交给系统打开。
- `./`、`../` 等相对路径按当前 Markdown 文件所在目录解析。

包含空格或中文的本地路径建议使用尖括号：

```md
[原型文件](<./App原型与截图/原型.html>)
```

相对路径不存在时，HorseMD 会提示而不是静默跳转。出于安全原因，未知或危险协议不会交给操作系统执行。

## 插入附件

桌面端可以通过“文件 → 插入附件…”或命令面板选择 PDF、DOCX、ZIP、音频等普通文件。当前 Markdown 必须先保存，因为附件目录需要以文档位置为基准。

HorseMD 会：

1. 在文档同级创建 `assets/` 文件夹。
2. 将附件复制进去，重名时生成不冲突的文件名。
3. 在当前光标处插入普通 Markdown 链接。

```md
[report.pdf](<assets/report.pdf>)
```

附件只作为链接，不在编辑器内预览。移动文档时应一起移动 `assets/`。移动端暂不提供附件复制入口，可使用系统分享功能处理文件。
